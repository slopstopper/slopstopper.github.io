Coherent doesn't mean it's correct. Completed doesn't mean it's been checked. Confidence doesn't mean it has provenance. It looks like it works — but does it have the right effect?

slopstopper is a set of tools that catch slop in code. Each stands alone, and any combination of them works together as one system — a plumb line for software: true vertical, by measurement, not by vibes.

§01the enemy

Named

Slop is coherence without provenance.

Fluent, plausible, confident output — unmoored from anything that checked it. It sounds finished, looks measured, and reads as decided.

Coherence used to be a signal: if a document held together, someone had probably thought about it. Generative tools broke that. Now everything holds together, including nonsense. slopstopper puts the missing signal back — as something a machine can check.

§02in use

Three tools · one system

Three concerns that don't overlap.

Honesty of claims, economics of effort, location of state. Each tool measures exactly one — and defers on the other two.

§03the recursion

Applied to itself

The rule of the house: if a discipline is too heavy to follow while building the tool that states it, the discipline is wrong.

plumb-line's audit skill was run against plumb-line and produced a remediation plan it then had to survive. recursive-spine's issues and milestones existed before its first commit; its labels were stamped by its own bootstrap; its deferrals age on its own digest. Change the rule and leave the record alone.

§04the register

Failure modes

What goes wrong when fluent tools meet real work.

R1Laundered uncertainty
A mock value gets used as if it were measured. It ends up in a dashboard, then a decision, and nothing along the way marks it as a guess.
R2Self-graded homework
The tool that made the claim is the same tool that checks it.
R3Prose-ledger rot
Work tracked in a document drifts from reality almost immediately. Merges drop rows and the file still reads fine.
R4Silent deferral
A “later” that isn't written down anywhere usually just doesn't happen.
R5Economics slop
An expensive model running work a cheap one could do — or a cheap one running work it can't, and having to redo it.
R6Overstated maturity
Everything gets called “production-ready,” including things that aren't.
§05the plan

Expanding

More will follow.

Three seams today, deliberately disjoint. The frame has room for more — and the vocabulary is pinned so the same word can't mean three things.

AXIS 01

Honesty

Are the claims true, and do they carry where they came from?

AXIS 02

Economics

Is the right model doing the work, at the right cost?

AXIS 03

State

Does the record of the work live where it can be queried?

Where the seams meet, the shared vocabulary is pinned in the marketplace — one owner per term, the others defer or alias. New instruments join on that condition, or not at all.

§07honestly

Status, honestly

None of this stops you from shipping slop. It just makes it visible, attributable, and slightly embarrassing.

From live practice in a few real projects, written up as practice reports — not benchmarks. We would love to tell you these tools are battle-tested; our own CI fails any README containing that phrase.

Instruments against synthetic coherence. Extracted from live practice, applied to themselves, and honest about what they don't yet do.
